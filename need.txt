지금까지 내용 + 양방향 요구까지 반영해서, 제미나이에게 그대로 줄 **최종 브리핑 문구**만 정리해 줄게.

***

### 제미나이에게 줄 최종 요청문

맥과 리눅스 사이에서 Raspberry Pi 400이 **양방향 KVM 브리지** 역할을 하도록 만들고 싶다. 아래 요구사항에 맞춰 전체 설계와 코드를 작성해줘.

***

#### 0. 현재 상태

- 하드웨어  
  - Raspberry Pi 400 (Raspberry Pi OS Lite, GUI 없음).  
  - Mac ↔ Pi 400: USB‑C 가젯 이더넷(usb0)으로 이미 연결, Mac에서 Pi로 SSH 접속 가능.  
  - Pi 400 내장 키보드/마우스 정상 동작.

- 네트워크  
  - Mac ↔ Pi는 usb0 대역(예: 192.168.42.x)에서 통신 가능.  
  - 리눅스 PC는 아직 Pi와 물리 연결만 계획 단계 (USB 케이블로 직접 연결 예정).

***

#### 1. 최종 목표 (양방향 KVM 브리지)

Raspberry Pi 400을 가운데 두고:

- 한쪽은 **Mac**, 다른 한쪽은 **리눅스 PC**.  
- 양쪽 모두에 대해 Pi가 **USB 키보드·마우스 장치(HID 가젯)** 로 보이게 해서,  
- 최종적으로는 다음이 가능해야 한다:

1) 맥 키보드/마우스로 리눅스를 조작  
2) 리눅스 키보드/마우스로 맥을 조작  
3) 선택적으로, Pi 400 내장 키보드/마우스를 소스로 써서 맥/리눅스 어느 쪽이든 조작

즉, **양방향**으로:

- Mac 입력 → Pi → USB HID → 리눅스  
- Linux 입력 → Pi → USB HID → Mac  

이 모두 가능한 구조를 설계하고 구현해 달라.

***

#### 2. 물리 연결 가정

구체적인 연결은 다음 형태를 가정해도 좋다 (필요하면 더 나은 구성을 제안해줘):

- Mac ↔ Pi:  
  - 이미 존재하는 **USB‑C 가젯 이더넷(usb0)** 채널 (네트워크/제어용).  

- Pi ↔ 리눅스 PC:  
  - 별도의 USB 케이블 (Pi를 리눅스에 **USB HID 장치**로 보이게).  

- (선택) Pi ↔ Mac 쪽 HID:  
  - 가능하다면, Pi가 맥에 대해서도 **USB HID 가젯**으로 잡히도록 두 번째 경로를 구성해서,  
  - 리눅스에서 온 입력을 맥으로 보낼 수 있게 해 달라.  
  - 만약 하드웨어/OS 제약으로 맥 쪽을 HID 가젯으로 만들기 어렵다면, **네트워크 기반(Barrier 등) 대체 방안**도 함께 제시.

***

#### 3. 해야 할 일 (설계 + 코드)

##### 3‑1. Pi 400에서 HID 가젯 구성 (Mac·Linux 양쪽 대상으로)

1) Pi를 리눅스 PC에 대해 USB HID(키보드 + 마우스)로 보이게 하는 설정:

- Raspberry Pi OS Lite(Bookworm)에서 `dwc2` + `libcomposite` + HID 가젯을 사용해,  
- 리눅스 PC에 Pi를 USB로 연결했을 때:  
  - `lsusb`, `dmesg`에서 “USB Keyboard/Mouse”로 보이도록.  
- `/boot/config.txt`, `/boot/cmdline.txt` 예시와 함께,  
- `setup_hid_gadget_linux.sh` 같은 configfs 스크립트 제공:
  - `/sys/kernel/config/usb_gadget/pi_kvm_linux` 생성  
  - idVendor/idProduct/strings  
  - `functions/hid.usb0` = 키보드, `functions/hid.usb1` = 마우스  
  - 표준 부트 HID report descriptor 포함  
  - `UDC` 설정으로 가젯 활성화.

2) 가능하다면, 맥에 대해서도 Pi가 USB HID 가젯으로 보이게 하는 경로를 설계:

- 예: `setup_hid_gadget_mac.sh` 또는 하나의 스크립트에서 타겟을 선택 가능하게.  
- macOS에서 Pi를 USB로 연결했을 때도 “외부 키보드/마우스”로 잡히게 할 수 있는지 조사하고,  
- 가능할 경우 설정 예시와 제약사항 정리.

3) 이미 Mac ↔ Pi 사이에는 `g_ether`(usb0)가 존재하므로,  
- `g_ether` + HID를 한 가젯으로 묶는 복합 가젯 구조 vs  
- 이더넷 가젯과 HID 가젯을 용도별로 분리하는 구조  
중 어떤 것이 현실적인지 비교하고, 권장안을 선택해 설계.

##### 3‑2. HID 이벤트 서버/클라이언트 (네트워크 ↔ /dev/hidg*)

Pi 상에서 동작하는 **HID 브리지 서버**를 설계해줘.

- 언어: Python 우선.  
- 역할:
  - 네트워크(TCP/HTTP/WebSocket)로 **양쪽(Mac, Linux)에서 들어오는 키/마우스 이벤트**를 받음.  
  - 받은 이벤트를 `/dev/hidg*`에 써서, 반대편 컴퓨터에 USB 입력으로 전달.  

요구사항:

- HID 포맷:
  - 키보드: 표준 부트 키보드 리포트 (modifier + 6 key slots 등).  
  - 마우스: 버튼 비트 + dx/dy + wheel.  

- API(예시):  
  - `POST /target/<mac|linux>/key`  
    - Body: `"text": "hello"` 또는 `"keys": ["LEFTCTRL", "C"]`  
  - `POST /target/<mac|linux>/mouse`  
    - Body: `{ "dx": 10, "dy": -5, "buttons": ["left"], "scroll": 0 }`  

- 서버는 “타겟(맥/리눅스)”을 선택해서 해당 쪽 HID 가젯으로 리포트를 써야 한다.  
  - 예: `/dev/hidg0_mac`, `/dev/hidg0_linux` 등, 또는 configfs로 분리된 디바이스.

##### 3‑3. 입력 소스 정의: Mac 키보드/마우스 + Pi 400 내장 키보드/마우스

입력 소스는 두 종류를 모두 지원하도록 설계해줘.

1) **Mac 입력 소스**  
   - macOS에서:
     - 글로벌 키보드/마우스 이벤트를 캡처해서,  
     - Pi HID 서버로 실시간 전송하는 클라이언트 프로그램 설계.  
   - 최소 버전:
     - CLI로 호출해서 텍스트/마우스 이벤트를 전송.  
   - 확장 버전(선택):
     - Quartz 이벤트 탭 등을 이용한 글로벌 후킹으로, 실제 마우스 움직임/키 입력을 실시간 전달.

2) **Pi 400 내장 키보드/마우스 입력**  
   - Pi 400 내부 키보드·마우스 이벤트를 읽어서 HID로 포워딩:  
     - `/dev/input/by-id/usb-Raspberry_Pi_Internal_Keyboard-...` 등에서 이벤트를 읽어,  
     - HID 리포트로 변환해 타겟(맥/리눅스)에 전달.  
   - 참고 구현으로 `pi400kb` 프로젝트를 활용해도 좋음.  
   - 모드 예시:
     - “소스 = Mac” / “소스 = Pi 키보드/마우스”/ “혼합” 모드를 설정으로 선택 가능하게.

##### 3‑4. 양방향 제어 시나리오

구체적인 시나리오 두 가지를 완성해줘:

1) 맥 → 리눅스  
   - 맥에서 키/마우스 사용 → 클라이언트 → Pi HID 서버 → Pi의 HID 가젯 → 리눅스에 입력.

2) 리눅스 → 맥  
   - 리눅스에서 키/마우스 사용:
     - 가능하다면 리눅스에서도 클라이언트(또는 글로벌 후킹)를 두고,  
     - Pi HID 서버에 이벤트를 보내서,  
     - Pi의 맥용 HID 가젯으로 맥에 입력.  
   - 맥 쪽이 USB HID로 받기 어려우면:
     - Pi ↔ Mac 사이에 Barrier/Synergy 같은 네트워크 기반 입력 공유를 사용하고,  
     - “리눅스 → 맥” 방향은 그 방식으로 대체하는 등 실용적인 대안을 제시.

각 시나리오에 대해:

- 필요한 가젯 디바이스,  
- 호출할 API 예시,  
- 테스트 절차(`curl`로 문자 전송, 간단한 키 조합 테스트 등)를 문서로 정리해줘.

##### 3‑5. 자동화 및 배포

- Pi 부팅 시 자동으로:
  - HID 가젯 설정 스크립트 실행,  
  - HID 서버(Python) 실행,  
  - (필요하다면) Pi 400 키보드 이벤트 포워더 실행.  
- 이를 위한 `systemd` 서비스 유닛 파일들을 작성해줘.  
- “처음부터 끝까지 한 번에 따라할 수 있는” 설치/설정 가이드를 작성해줘:
  - 파일 배치 위치,  
  - `chmod`, `systemctl enable`,  
  - Mac/리눅스 쪽 클라이언트 설치·실행 방법까지.

***

#### 4. 한 줄 요약

이미 Mac ↔ Raspberry Pi 400은 USB 가젯 이더넷으로 SSH 연결이 되어 있다. 이제 Pi 400을 Mac과 Linux 사이의 **양방향 KVM 브리지**로 만들어서, Mac·Linux 어느 쪽 키보드/마우스를 쓰든, Pi를 통해 상대 쪽에 USB HID 입력으로 전달할 수 있게 하고 싶다. 위 요구사항을 만족하는 아키텍처, 가젯 설정 스크립트, HID 서버(Python), Mac/Linux 클라이언트, systemd 자동화까지 모두 설계하고 코드를 작성해줘.
